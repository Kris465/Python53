### **Тема 3.1: Менеджер пакетов PIP**

#### **1. Введение: Что такое PIP?**
*   **PIP (Pip Installs Packages)** — стандартный **менеджер пакетов** (система управления программными библиотеками) для языка Python.
*   **Основная задача:** Упрощение процесса установки, обновления, удаления и управления дополнительными библиотеками (пакетами), которые не входят в стандартную библиотеку Python.
*   Автоматически разрешает **зависимости** (устанавливает другие пакеты, необходимые для работы требуемого пакета).

#### **2. Что такое PyPI (Python Package Index)?**
*   **PyPI (Python Package Index)** — центральный **онлайн-репозиторий** (хранилище) пакетов Python. Это огромная коллекция открытого программного обеспечения.
*   **Принцип работы:** По умолчанию `pip` ищет и скачивает пакеты именно из PyPI. Можно представить это как "магазин приложений" для Python.
*   Каждый пакет на PyPI имеет свою страницу с описанием, версиями, инструкциями и зависимостями.

#### **3. Основные команды PIP**

##### **`pip install` — Установка пакетов**
*   **Установка последней версии:**
    ```bash
    pip install requests
    ```

*   **Установка конкретной версии:**
    ```bash
    pip install django==4.2.0
    ```

*   **Обновление пакета до последней версии:**
    ```bash
    pip install --upgrade requests
    ```

*   **Установка пакета из локального файла:**
    ```bash
    pip install ./some_package.whl
    ```

*   **Установка зависимостей из файла (см. `pip freeze`):**
    ```bash
    pip install -r requirements.txt
    ```

##### **`pip list` — Просмотр установленных пакетов**
*   Выводит список всех пакетов, установленных в текущем окружении Python, вместе с их версиями.
    ```bash
    pip list
    ```
    *Пример вывода:*
    ```
    Package    Version
    ---------- -------
    numpy      1.24.3
    pandas     2.0.3
    pip        23.1.2
    requests   2.31.0
    ```

##### **`pip freeze` — Фиксация версий пакетов**
*   Выводит список установленных пакетов в формате `package==version`, который **удобен для сохранения в файл**.
*   **Ключевое применение — создание файла зависимостей `requirements.txt`** для воспроизведения окружения (например, для другого разработчика или на сервере).
    ```bash
    pip freeze > requirements.txt
    ```
    Содержимое файла `requirements.txt`:
    ```
    numpy==1.24.3
    pandas==2.0.3
    requests==2.31.0
    ```

##### **`pip uninstall` — Удаление пакета**
*   Удаляет указанный пакет из системы.
    ```bash
    pip uninstall pandas
    ```
    *Будет запрошено подтверждение.*

#### **4. Поиск пакетов**
*   **Через веб-интерфейс PyPI:** [https://pypi.org](https://pypi.org). Лучший способ для поиска и изучения пакетов (чтение описаний, документации).
*   **Через командную строку pip (устаревший способ):**
    ```bash
    pip search "поисковый запрос"
    ```
    *Примечание: Эта команда часто отключена из-за нагрузок на сервер PyPI. Рекомендуется использовать сайт.*

#### **5. Важные практические аспекты**
*   **Права доступа:** В macOS/Linux при установке пакетов глобально в системный Python может потребоваться `sudo`. **Лучшая практика — использование виртуальных окружений (`venv`)**.
*   **Виртуальные окружения (`venv`):** Позволяют создавать изолированные среды для разных проектов, чтобы избежать конфликтов версий пакетов. `pip` работает внутри такого окружения.
    ```bash
    # Создание окружения
    python -m venv my_env
    # Активация (Linux/macOS)
    source my_env/bin/activate
    # Активация (Windows)
    my_env\Scripts\activate
    # После активации все действия pip будут происходить внутри этого окружения
    ```
*   **`pip show <имя_пакета>`:** Полезная команда для получения подробной информации об установленном пакете (версия, расположение, зависимости, автор).
    ```bash
    pip show requests
    ```

---

### **Ключевые выводы:**
1.  **PIP** — инструмент №1 для управления внешними библиотеками Python.
2.  **PyPI** — главный источник этих библиотек.
3.  **Базовая цепочка работы:** Поиск на pypi.org → Установка через `pip install` → Фиксация версий в `requirements.txt` с помощью `pip freeze`.
4.  **Файл `requirements.txt`** — обязательный атрибут любого проекта для обеспечения воспроизводимости его окружения.
5.  **Использование виртуальных окружений (`venv`)** — стандартный и профессиональный подход к изоляции зависимостей проекта.
